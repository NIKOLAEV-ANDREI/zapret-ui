# Zapret UI

Графический менеджер для текущей папки `zapret-fix`.

## Запуск

Готовый файл после сборки:

```text
C:\zapret-fix\zapret.exe
```

Приложение запрашивает права администратора. Это нужно для управления Windows Service, `WinDivert`, процессами `winws.exe`, реестром и сетевыми настройками.

## Ярлык

Ярлык создается на рабочем столе:

```text
Zapret.lnk
```

Если нужно пересоздать ярлык:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\ui\create-shortcut.ps1
```

## Сборка без .NET SDK

На этой машине нет .NET SDK, поэтому сборка сделана через системный `csc.exe` из .NET Framework:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\ui\build.ps1
```

## Сборка через .NET SDK

Файл `zapret.csproj` тоже добавлен. Если установить .NET SDK, можно собрать обычным способом:

```powershell
dotnet build .\ui\zapret.csproj
```

## Что реализовано

- выбор и запуск стратегии из `general*.bat`;
- установка выбранной стратегии как службы `zapret`;
- остановка `winws.exe`;
- удаление служб `zapret`, `WinDivert`, `WinDivert14`;
- статус службы, процесса, IPSet, Game Filter и авто-проверки обновлений;
- переключение Game Filter;
- переключение IPSet `loaded -> none -> any -> loaded`;
- обновление IPSet;
- проверка обновлений;
- ручная подготовка обновления `hosts`;
- диагностика конфликтов;
- встроенный обычный тест стратегий через HTTP/TLS/Ping;
- отмена встроенного теста с остановкой текущего `winws.exe` и прерыванием текущего `curl`;
- группировка результатов теста по стратегиям;
- цветная сводка по каждому конфигу и список лучших конфигов после завершения;
- сохранение результатов встроенного теста в `utils\test results`;
- запуск существующих тестов `utils\test zapret.ps1` как резервный вариант;
- редактор пользовательских списков и целей тестов;
- создание ярлыка на рабочем столе.

## Темы

Интерфейс поддерживает светлую и темную тему. Для темной темы отдельно стилизованы стандартные WPF-контролы: списки, выпадающие списки, текстовые поля, таблицы, строки и заголовки таблиц.

## Важные пояснения

`general (ALT5).bat` не рекомендуется самим исходным набором файлов. Он использует более агрессивные режимы `syndata,multidisorder`, меньше ограничивает фильтрацию и может сильнее задевать обычный трафик.

`Game Filter` не применяется поверх уже запущенного процесса. Он меняет флаг, который подставляется в аргументы `winws.exe` при следующем запуске стратегии. После изменения Game Filter нужно перезапустить конфиг или переустановить службу.

## Если основной exe занят

Если `zapret.exe` запущен с правами администратора, обычная сборка не сможет заменить файл. Закройте окно приложения и выполните:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\ui\apply-ui-update.ps1
```
